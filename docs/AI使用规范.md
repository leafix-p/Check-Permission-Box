# AI 使用规范

1. 提交前必须进行代码规范及功能检查，提交时必须经过人为确认
2. 对功能进行扩充或修改，必须补充相关的开发文档
3. 代码生成必须遵循基本语法
4. 生成方法必须带方法功能使用注释，代码行间注释必须注明到每一个阶段
5. 每个文档或代码文件内容结尾必须带空行
6. 界面显示的文字字符串禁止硬编码
7. 对文档进行修改时，必须同步更新根目录 [`README.md`](../README.md) 中的项目描述、功能列表和文件结构等必要信息
8. 生成的 Markdown 表格必须遵循以下格式规范，确保 JetBrains 系 IDE 不报错：
   - 每一列在分隔行（`---`）中至少使用 3 个 `-`
   - 所有行的列数必须一致
   - 单元格内容与 `|` 之间保留至少 1 个空格
   - 分隔行的 `-` 数量建议对齐到该列最宽内容的长度
   - 行末 `|` 之后禁止有尾随空格
