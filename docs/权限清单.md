# 权限清单

## 权限名称

MANAGE_EXTERNAL_STORAGE

## 权限字符串值

android.permission.MANAGE_EXTERNAL_STORAGE

## 权限用途

允许应用访问设备上的所有文件（外置存储全部文件读写权限），无需用户逐个文件夹授权。

## 适用系统 API 范围

Android 11 (API 30) 及以上

## 权限申请方法

1. 在 AndroidManifest.xml 中声明权限：
   `<uses-permission android:name="android.permission.MANAGE_EXTERNAL_STORAGE" />`

2. 在代码中检查权限是否已授予：
   ```kotlin
   if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
       Environment.isExternalStorageManager()
   }
   ```

3. 若未授权,跳转系统设置页面引导用户授权：
   ```kotlin
   val intent = Intent(Settings.ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION).apply {
       data = Uri.parse("package:${context.packageName}")
   }
   startActivity(intent)
   ```

4. 使用 `ActivityResultLauncher` 或监听 `onResume` 生命周期,在用户返回后重新检查权限状态。

## CAMERA

- **权限字符串值：** android.permission.CAMERA
- **权限用途：** 允许应用拍摄照片和视频
- **适用系统 API 范围：** 所有版本
- **权限申请方法：** 运行时权限，通过 `ActivityResultContracts.RequestPermission()` 弹出系统对话框申请

## 注意事项

- MANAGE_EXTERNAL_STORAGE 在 Google Play 商店有严格限制，仅限文件管理器、备份、杀毒等特定类别应用使用。
- POST_NOTIFICATIONS、READ_MEDIA_IMAGES 为 Android 13 新增运行时权限，低版本系统会自动跳过。
- BLUETOOTH_CONNECT 为 Android 12 新增运行时权限，低版本系统会自动跳过。
- 本示例仅用于学习和展示 Android 权限申请机制。
