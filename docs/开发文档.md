# 开发文档

## 项目概述

开发目标是需要编写所有的权限申请逻辑，并且每个权限都有一个开关，检查权限是否被允许，如果被允许则打开状态，不允许则关闭。
从关闭->打开：尝试申请权限，如果成功则打开，失败则恢复关闭状态。

## 架构设计

### 分层解耦

```
┌─────────────────────────────────────────────────┐
│  PermissionDef  (model/PermissionDef.kt)         │
│  数据层：权限名称、描述、API要求、检查/申请逻辑      │
│  ┌─ MANAGE_EXTERNAL_STORAGE                      │
│  └─ (未来新增的其他权限...)                        │
└──────────────┬──────────────────────────────────┘
               │ 传入数据
               ▼
┌─────────────────────────────────────────────────┐
│  PermissionItem  (ui/permission/PermissionItem.kt)│
│  UI层：通用渲染组件，数据驱动                      │
│  - 自动判断 API 兼容性（不满足则禁用）              │
│  - Switch 状态管理                                │
│  - ActivityResultLauncher 处理授权回调             │
│  - onResume 生命周期监听刷新权限状态                │
└──────────────┬──────────────────────────────────┘
               │ 遍历 ALL_PERMISSIONS
               ▼
┌─────────────────────────────────────────────────┐
│  PermissionList  (MainActivity.kt)               │
│  页面编排：设备信息 + 权限列表                     │
└─────────────────────────────────────────────────┘
```

### 核心优势

1. **数据驱动**：UI 完全由 `PermissionDef` 驱动，新增权限无需修改界面代码
2. **关注点分离**：权限数据（PermissionDef）与 UI 渲染（PermissionItem）解耦
3. **开闭原则**：新增权限只需添加数据定义，对已有代码零侵入

## 如何新增一个权限

### 三步完成

#### 第一步：声明权限

在 [`AndroidManifest.xml`](../app/src/main/AndroidManifest.xml) 中添加 `<uses-permission>`：

```xml
<uses-permission android:name="android.permission.xxx" />
```

#### 第二步：添加字符串资源

在 [`strings.xml`](../app/src/main/res/values/strings.xml) 中新增权限名称、描述和版本提示：

```xml
<string name="xxx_permission">权限名称</string>
<string name="xxx_permission_desc">权限用途描述</string>
<string name="xxx_required_api">需要 Android X (API X) 及以上</string>
```

#### 第三步：注册权限定义

在 [`PermissionDef.Companion`](../app/src/main/java/com/leafix/checkpermissionbox/model/PermissionDef.kt) 中添加定义，并加入 `ALL_PERMISSIONS` 列表：

```kotlin
val SAMPLE_PERMISSION = PermissionDef(
    nameResId = R.string.xxx_permission,
    descriptionResId = R.string.xxx_permission_desc,
    requiredApiTextResId = R.string.xxx_required_api,
    minSdk = Build.VERSION_CODES.XXX,
    checkPermission = { ctx ->
        /* 权限检查逻辑，接收 Context 参数，返回 Boolean */
        true
    },
    requestMethod = RequestMethod.RuntimePermission("android.permission.XXX")
)

val ALL_PERMISSIONS: List<PermissionDef> = listOf(
    MANAGE_EXTERNAL_STORAGE,
    SAMPLE_PERMISSION   // ← 新增权限追加在此
)
```

完成以上三步，权限条目会自动出现在首页列表中，API 不满足时自动禁用。

## 权限清单编写规范

对于权限清单的编写，需要遵循以下规则：

1. 标题为权限名称，
2. 解释中第一行为权限的字符串值，
3. 第二行为权限的用途，
4. 第三行为权限的适用系统api范围，
5. 第四行写上权限的申请方法
